# 05. Gitの使い方

チーム開発では、いきなり本体に変更を入れません。

自分用の作業場所を作ってから、変更します。

## 1. 最新状態を取る

```bash
git pull
```

## 2. 自分の作業ブランチを作る

すわの場合：

```bash
git switch -c suwa/product-list
```

櫻井の場合：

```bash
git switch -c sakurai/product-card
```

## 3. 作業する

ファイルを直します。

## 4. 何を変更したか見る

```bash
git status
```

## 5. 変更を登録する

```bash
git add .
```

## 6. 変更に名前を付ける

```bash
git commit -m "商品カードの見た目を調整"
```

## 7. GitHubへ送る

```bash
git push
```

初回だけ、ターミナルに出た長いコマンドをそのまま実行することがあります。

## コミットメッセージの例

```text
商品一覧画面を追加
商品カードの色を調整
商品出品フォームを作成
店舗管理画面の表示を修正
売り切れ表示を追加
```

## やらないこと

```bash
git push --force
```

これは基本使いません。

```bash
git reset --hard
```

これも勝手に使いません。必要ならチームに相談してください。

## 作業前に確認すること

- 今いるブランチは正しいか
- `git pull` したか
- 変更したファイルを説明できるか

## 作業後に確認すること

- `npm run build` が通るか
- `npm run lint` が通るか
- 画面が壊れていないか
