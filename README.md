# MineTenant フロント練習用リポジトリ 手順書

## 0. この手順書について

この手順書は、MineTenantのフロント担当者が、React・TypeScript・Viteを使った画面制作に慣れるためのものです。

対象者は、フロントエンド初学者を想定しています。

最初から全部を完璧に理解する必要はありません。

大事なのは、次の流れを繰り返すことです。

```text
動かす
↓
少し変える
↓
壊れる
↓
原因を見る
↓
直す
↓
説明できるようにする
```

今回の目的は、本番開発に入る前に、練習用リポジトリでMineTenantの基本画面を触りながら、Reactの画面作りに慣れることです。

---

## 1. 今回作るもの

この練習用リポジトリでは、MineTenantのフロント画面の練習をします。

MineTenantは、Web上のCtoCサイトとMinecraftを連携させるサービスです。

フロント担当は、主にWeb画面を作ります。

今回の練習リポジトリでは、次の画面を扱います。

```text
商品一覧画面
商品詳細画面
商品出品画面
店舗管理画面
購入の流れ
店舗ポイント・店舗レベル表示
```

本番のようにバックエンドやMinecraftとはまだ接続しません。

最初は仮の商品データを使って、画面だけで動く状態を確認します。

## 20. vscode AIを使うときのルール

vscode AIは便利ですが、丸投げしないでください。

AIが作ったコードを理解できないまま入れると、あとで直せなくなります。

良い頼み方です。

```text
ProductCard.tsx の商品カードに、在庫が0のとき「売り切れ」と表示する処理を追加してください。
初心者にも分かるように、どこを変更したか説明してください。
```

```text
StoreDashboard.tsx の店舗レベル表示を見やすくしたいです。
既存の構成を大きく変えずに、初心者でも理解できる形で修正してください。
```

```text
このエラーの原因を初心者にも分かるように説明してください。
勝手に大きく作り直さず、最小限の修正だけ教えてください。
```

悪い頼み方です。

```text
全部いい感じにして
```

```text
本番レベルに作り直して
```

```text
好きに全部修正して
```

AIに頼むときは、必ず以下を伝えてください。

```text
どのファイルを変更したいか
何を追加したいか
どのくらい大きく変更していいか
初心者向けに説明してほしいこと
```

## 25. エラーが出たとき

エラーが出たら、まず慌てないでください。

見る場所は以下です。

```text
ターミナル
ブラウザ画面
vscodeのProblemsタブ
```

よく使う確認コマンドです。

```bash
npm run dev
```

開発サーバーを起動します。

```bash
npm run build
```

本番用に問題なくビルドできるか確認します。

```bash
npm run lint
```

コードの書き方に問題がないか確認します。

分からないときは、vscode AIに以下のように聞きます。

```text
このエラーの原因を初心者にも分かるように説明してください。
どのファイルのどの部分を直せばいいか教えてください。
勝手に大きく作り直さず、最小限の修正だけにしてください。
```

## 26. よくあるエラーと考え方

### 画面が表示されない

確認することです。

```text
npm install をしたか
npm run dev をしたか
表示されたURLを開いているか
ターミナルにエラーが出ていないか
```

### 文字を変えたのに画面が変わらない

確認することです。

```text
保存したか
編集しているファイルが正しいか
ブラウザを更新したか
別の場所に同じ文字がないか
```

### TypeScriptのエラーが出る

TypeScriptは、データの形が違うとエラーを出します。

例です。

```text
価格は数字なのに、文字として入れている
在庫は数字なのに、空文字になっている
必要な項目が足りない
```

分からない場合は、エラー文をコピーしてvscode AIに聞いてください。

### Gitでエラーが出る

確認することです。

```text
git pull したか
誰かと同じファイルを変更していないか
commit していない変更が残っていないか
```

困ったら、勝手にリセットせず、チームに相談してください。
